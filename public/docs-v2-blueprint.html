Sambla v2.0 — Full Implementation Blueprint

Production-grade upgrade: multi-intent orchestration, conversion tracking, lead capture, WooCommerce deep integration, tenant customization, analytics engine. Generated 29 March 2026.

1. Executive Summary

This blueprint transforms Sambla from a chat responder into a measurable sales assistant and lead-generation engine. The upgrade adds 12 major modules while preserving the existing architecture (47 tables, 29 models, multi-tenant isolation).

Key additions: multi-intent orchestration, product card UX with add-to-cart, WooCommerce checkout attribution, conversation outcome engine, intelligent lead capture, tenant conversation policies, advanced recommendations, RAG retrieval memory, complexity-based model routing, human handoff, full analytics/attribution, and admin insights.

Approach: Incremental — no destructive rewrites. New tables/services alongside existing ones. Feature-flagged rollout per tenant.

5. New/Changed Backend Services

5.1 IntentOrchestratorService (NEW — core)

class IntentOrchestratorService
{
    /**
     * Analyze message and return orchestration plan with multiple intents.
     *
     * @return OrchestratorPlan {intents: DetectedIntent[], pipelines: PipelineTask[], context: array}
     */
    public function plan(string $message, Conversation $conversation, Bot $bot): OrchestratorPlan;

    /**
     * Execute orchestration plan and merge results.
     *
     * @return OrchestratorResult {productContext, orderContext, knowledgeContext, leadContext, handoffContext, products[]}
     */
    public function execute(OrchestratorPlan $plan, int $botId): OrchestratorResult;
}

// DetectedIntent DTO
class DetectedIntent
{
    public string $name;           // order_lookup, product_search, category_recommendation, knowledge_query, lead_intent, quote_intent, handoff_intent, greeting, thanks
    public float $confidence;      // 0.0–1.0
    public array $entities;        // extracted: order_number, product_name, category, etc.
    public int $priority;          // execution order
}

// OrchestratorPlan DTO
class OrchestratorPlan
{
    public array $intents;         // DetectedIntent[]
    public array $pipelines;       // PipelineTask[] — ordered execution
    public bool $needsClarification;
    public ?string $clarificationQuestion;
    public float $complexityScore;
}

Key difference from current: Current IntentDetectionService returns booleans (is_order_query, is_product_search, etc.) and controller uses first-match-wins. New orchestrator returns ALL matched intents with scores and composes execution plan.

class LeadOpportunityScorer
{
    /**
     * Score the current conversation for lead capture opportunity.
     * Returns 0-100 score + recommended trigger reason.
     *
     * Signals scored:
     * - Has buying intent but not ready (+20)
     * - Asked for quote/callback/consultation (+30)

     * - Very short conversation (-20)

5.6 ComplexityScoringService (NEW)

class ComplexityScoringService
{
    /**
     * Score message complexity to determine model tier.
     * Replaces simple word-count logic in ChatModelRouter.
     *
     * Factors:
     * - intent_count: more intents → higher complexity
     * - ambiguity: multiple possible interpretations
     * - follow_up_dependency: references previous messages
     * - comparison_request: "care e mai bun" type queries
     * - calculation_needed: quantities, prices, areas
     * - policy_lookup: delivery, returns, warranty
     * - emotional_cues: frustration, urgency, complaint
     * - lead_opportunity: high-value interaction
     * - input_quality: typos, noisy transcription
     * - orchestration_needed: products + knowledge simultaneously
     *
     * Returns 0-100 score.
     * < 30 → fast model
     * 30-70 → fast or smart (cost-aware)
     * > 70 → smart model
     */
    public function score(string $message, array $intents, Conversation $conversation): int;
}

class InsightGeneratorJob implements ShouldQueue
{

     * Generate actionable insights per tenant.
     * Runs daily or on-demand.
     *

     * - "kb_gap": many queries hit 0 knowledge results → suggest adding content

                this.trackEvent('variation_required_redirect', { product_id });
                window.open(product.permalink, '_blank');
                atcBtn.textContent = '→ Vezi produs';
            } else {
                atcBtn.textContent = data.error_message || 'Eroare';
            }
        }
    } catch (err) {
        atcBtn.textContent = 'Eroare rețea';
    }

    setTimeout(() => { atcBtn.textContent = '🛒 Adaugă'; atcBtn.disabled = false; }, 3000);

17. Security/Privacy

  • GDPR: Lead capture requires explicit consent checkbox. Leads must be exportable/deletable per tenant.

  • Attribution: No cross-tenant attribution possible. All queries scoped to tenant_id.

21.2 IntentOrchestratorService Skeleton

// app/Services/IntentOrchestratorService.php
class IntentOrchestratorService
{
    public function __construct(
        private IntentDetectionService $intentDetector,
        private ProductSearchService $productSearch,
        private RecommendationService $recommendations,
        private KnowledgeSearchService $knowledge,
        private OrderLookupService $orderLookup,
        private LeadOpportunityScorer $leadScorer,
        private HandoffService $handoff,
        private ConversationEventService $events,
    ) {}

    public function plan(string $message, Conversation $conversation, Bot $bot): OrchestratorPlan
    {
        // 1. Detect ALL intents with scores
        $intents = $this->intentDetector->detectAll($message);
        // Returns: [{name: 'product_search', confidence: 0.9, entities: {query: 'adeziv gresie'}},
        //           {name: 'order_lookup', confidence: 0.85, entities: {order_number: '1234'}}]

        // 2. Score lead opportunity
        $events = $this->events->getSessionEvents($conversation->id);
        $leadScore = $this->leadScorer->score($conversation, $intents, $events);
        if ($leadScore->value >= $leadScore->threshold) {
            $intents[] = new DetectedIntent('lead_opportunity', $leadScore->value / 100, [], 99);
        }

        // 3. Check handoff triggers
        if ($this->handoff->shouldOffer($conversation, $intents, $events)) {
            $intents[] = new DetectedIntent('handoff', 0.9, [], 100);
        }

        // 4. Sort by priority, build execution plan
        usort($intents, fn($a, $b) => $a->priority <=> $b->priority);

        // 5. Calculate complexity
        $complexity = app(ComplexityScoringService::class)->score($message, $intents, $conversation);

        return new OrchestratorPlan(
            intents: $intents,
            pipelines: $this->buildPipelines($intents),
            needsClarification: false,
            clarificationQuestion: null,
            complexityScore: $complexity,
        );
    }

    public function execute(OrchestratorPlan $plan, int $botId): OrchestratorResult
    {
        $result = new OrchestratorResult();

        foreach ($plan->pipelines as $pipeline) {
            match ($pipeline->name) {
                'order_lookup' => $result->orderContext = $this->executeOrderLookup($botId, $pipeline),
                'product_search' => $this->executeProductSearch($botId, $pipeline, $result),
                'recommendation' => $this->executeRecommendation($botId, $pipeline, $result),
                'knowledge' => $result->knowledgeContext = $this->executeKnowledge($botId, $pipeline),
                'lead_capture' => $result->leadContext = $this->executeLeadCapture($pipeline),
                'handoff' => $result->handoffContext = $this->executeHandoff($pipeline),
            };

            // Track pipeline execution
            $this->events->track('pipeline_executed', [
                'pipeline' => $pipeline->name,
                'duration_ms' => $pipeline->durationMs,
                'results_count' => $pipeline->resultsCount,
            ]);
        }

        return $result;
    }
}

21.4 Widget Event Tracking (JavaScript)

// In sambla-chat.js — event batch sender
class EventTracker {
    constructor(apiBase, channelId) {
        this.queue = [];
        this.apiBase = apiBase;
        this.channelId = channelId;
        this.flushInterval = setInterval(() => this.flush(), 5000);
    }

    track(eventName, properties = {}) {
        this.queue.push({
            event_name: eventName,
            properties,
            session_id: this.sessionId,
            visitor_id: this.visitorId,
            bot_id: this.botId,
            conversation_id: this.conversationId,
            idempotency_key: `${this.sessionId}:${eventName}:${properties.product_id || ''}:${Math.floor(Date.now()/60000)}`,
            occurred_at: new Date().toISOString(),
        });

        // Flush immediately for critical events
        if (['add_to_cart_click', 'lead_completed', 'session_ended'].includes(eventName)) {
            this.flush();
        }
    }

    async flush() {
        if (this.queue.length === 0) return;
        const events = [...this.queue];
        this.queue = [];

        try {
            await fetch(`${this.apiBase}/api/chatbot/${this.channelId}/events`, {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({ events }),
                keepalive: true, // survives page navigation
            });
        } catch (e) {
            this.queue.unshift(...events); // retry on next flush
        }
    }
}

        }
        foreach ($policy->requiredPhrases as $phrase) {
            $lines[] = "- Menționează când e relevant: \"{$phrase}\"";
        }
        foreach ($policy->businessRules as $rule) {
            $lines[] = "- REGULĂ DE BUSINESS: {$rule}";
        }

        $lines[] = "=== SFÂRȘIT STIL ===";
        return implode("\n", $lines);
    }
