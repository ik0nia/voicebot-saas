RAG Post-Implementation Audit

Sambla voicebot-saas · 1 Aprilie 2026 · All improvements implemented and validated

1 Executive Summary

All agreed RAG improvements have been implemented, tested, and validated. The system now features stored tsvector for faster FTS, HNSW ef_search tuning, conditional reranking, parent-child chunk retrieval, variable chunk overlap, language-aware search (Romanian + English), semantic product retrieval architecture, and a feedback loop mechanism. Price/stock/SKU have been removed from product embeddings. TurboQuant was NOT implemented (deferred as agreed).

5 Product Retrieval Architecture

Embedded (semantic identity)

  • Product name/title
  • Short description (stripped HTML)
  • Long description (truncated 1500 chars)
  • Categories (full path)
  • Attributes (key: value format)

NOT embedded (live/structured)

  • Price / sale price
  • Stock status / quantity
  • SKU
  • URLs / permalinks
  • Image URLs

Injected at query time from woocommerce_products table

11 TurboQuant Status

TurboQuant was NOT implemented. This was explicitly deferred as agreed. The decision was correct: the practical RAG improvements provide measurable quality gains at near-zero cost. TurboQuant's complexity and risk profile are not justified at current scale. It remains a LATER consideration if retrieval quality plateaus after all practical improvements are exhausted.
