2. CONCEPTUL LEAD vs OPPORTUNITY

Definiții clare:

LeadOpportunity
DefinițieConversație cu DATE DE CONTACT (email / telefon / nume)Conversație cu INTENȚIE dar FĂRĂ date de contact
Exemplu"ion@email.com vrea ofertă pentru polistiren"User a văzut 5 produse, a dat click pe 3, dar a plecat fără contact
SursăTabel `leads` (cu email/phone/name)Derivat din `conversations` + `chat_events` + `conversation_outcomes`
ScopFollow-up, CRM, conversieOptimizare: de ce nu a lăsat date? ce lipsea?
AcțiuneContactare, ofertăÎmbunătățire lead capture, UX, produse

2.1 Schema DB — Opportunities (NOU)

-- Opportunities sunt derivate din conversații, NU un tabel separat.
-- Se calculează dinamic sau se stochează ca outcome_type în conversation_outcomes.

-- Adăugăm coloane pe conversations:
ALTER TABLE conversations ADD COLUMN opportunity_score SMALLINT DEFAULT 0;
ALTER TABLE conversations ADD COLUMN is_opportunity BOOLEAN DEFAULT FALSE;
ALTER TABLE conversations ADD COLUMN opportunity_reasons JSONB;

-- Index:
CREATE INDEX idx_conversations_opportunity ON conversations (tenant_id, is_opportunity, created_at)
    WHERE is_opportunity = TRUE;

De ce nu tabel separat: O opportunity ESTE o conversație. Adăugăm metadata pe conversație, nu duplicăm date.

2.2 Când devine o conversație opportunity?

Reguli (evaluate de OutcomeEngine sau DeriveConversationOutcomes job):

1. products_returned > 0 (user a văzut produse)              +20
2. product_click > 0 (user a dat click pe produs)             +25
3. add_to_cart_success > 0 (user a adăugat în coș)            +35
4. messages_count >= 3 (conversație substanțială)             +10
5. duration >= 2 minute                                       +10
6. intent = product_search sau recommendation                  +15
7. NU are lead asociat (leads.conversation_id = NULL)          obligatoriu

Threshold: opportunity_score >= 30

Dacă NU are lead = opportunity.
Dacă ARE lead = nu mai e opportunity, e lead.

// database/migrations/2026_03_30_000002_add_opportunities_and_company.php

});

12. WORKFLOW COMPLET

USER INTERACȚIONEAZĂ CU CHATBOT
    │
    ├─ Conversație → chat_events (impression, click, ATC)
    │
    ├─ Lead prompt arătat?
    │   ├─ DA → user completează → Lead creat (status: new/qualified)
    │   └─ NU / refuzat → conversație se termină
    │
    ├─ session_ended trigger
    │   └─ DeriveConversationOutcomes job
    │       ├─ Calculează outcomes (product_discovered, add_to_cart, etc.)
    │       ├─ Calculează opportunity_score
    │       │   ├─ Score >= 30 ȘI fără lead → is_opportunity = TRUE
    │       │   └─ Are lead → is_opportunity = FALSE
    │       └─ Salvează pe conversation
    │
    ├─ Purchase webhook (WooCommerce)
    │   └─ AttributionService
    │       ├─ Match strict / probable / assisted
    │       └─ PurchaseAttribution creat + revenue tracked
    │
    └─ Dashboard afișează:
        ├─ KPIs: sesiuni, leads, opportunities, revenue
        ├─ Funnel: conversations → products → clicks → ATC → purchase
        ├─ Leads tab: cu date contact, scor, status
        ├─ Opportunities tab: fără contact, cu intent, drop-off point
        └─ Commerce tab: attribution breakdown, top produse, timp mediu
