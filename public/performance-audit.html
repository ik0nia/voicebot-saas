4. CALITATE RAG

Retrieval-Augmented Generation — unde se pierde calitatea

4 quick wins fara cod (doar config changes) care imbunatesc relevanta cu 15-20%.

🎯
Similarity Threshold: 0.55 → 0.62
Pragul curent (0.55) e prea permisiv — lasa rezultate cu relevanta slaba. La 0.62, creste precizia cu ~10-15% fara pierdere semnificativa de recall.
HIGH IMPACT 1 config change
⚖️
FTS Weight: 1.5 → 1.0
Full-text search are 50% mai multa influenta decat vector search. Asta biaseaza spre keyword matching in loc de intelegere semantica. Egalizarea la 1.0 ajuta queries conversationale.
MEDIUM IMPACT 1 config change
🔍
Reranking Zone: 0.50-0.78 → 0.58-0.85
Zona actuala e prea larga. Sub 0.58 reranking-ul nu salveaza rezultate slabe. Peste 0.78, chiar si rezultatele bune pot beneficia de reranking. Snippet 350 → 600 chars.
HIGH IMPACT 3 config changes
🔇
Disable Parent-Child Retrieval
Chunk-urile adiacente adaugate fara re-ranking dilueaza contextul cu continut potential irelevant. Dezactivare reduce zgomotul si imbunatateste raspunsurile.
MEDIUM IMPACT 1 config change
📏
Context Budget: 6000 → 8000 chars
6000 chars (1500 tokens) e strans pentru queries complexe. 8000 chars (2000 tokens) da mai mult spatiu fara depasirea bugetului total.
MEDIUM IMPACT 1 config change
🇷🇴
Enable LLM Query Rewrite (Romanian)
Rescrierea query-ului cu GPT-4o-mini inainte de cautare captureaza diminutive, sinonime si greseli de ortografie romanesti. Cost: ~80 tokens/query.
MEDIUM IMPACT 1 config change

6. COST AI

Eficienta LLM — $0.42/zi, 40% reducere posibila

Costurile sunt deja mici, dar exista potential de 40-50% reducere prin optimizari simple.

📊
Summary Threshold: 8 → 12
Sumarizarea conversatiei se activeaza la 8 mesaje. La 12, economisesti 40% din call-urile de summarization fara pierdere de context.
5 min
💰
Skip KB pentru greetings
85% din mesaje triggereaza knowledge search. Saluturile, confirmarile si multumirile nu au nevoie de RAG. Skip = -20% embedding calls.
2 ore
⚡
Response cache: 3min → 15min
Cache-ul de raspunsuri LLM (3 min TTL) e prea conservativ. La 15 min cu query normalization mai agresiva, hit rate creste de la 15% la 30%.
2 ore
🔄
Soft fallback: gpt-4o → gpt-4o-mini
Cand gpt-4o esueaza, sistemul trece pe Claude (12-25x mai scump). Mai bine fallback pe gpt-4o-mini (acelasi provider, mult mai ieftin).
1 ora
🗜️
Reduce policy prompt bloat
PromptBuilder injecteaza politica conversationala completa chiar si cand e default. Injectie conditionala = -150-300 tokens/mesaj.
1 ora
🌡️
Temperature per intent
FAQ si order lookup = factual (temp 0.3). Recomandari = creativ (temp 0.7). Temperature fix de 0.6 e suboptimal pentru ambele.
1 ora

8. WIDGET FRONTEND

Widget Chat — 94KB, memory leaks, no virtualization

📦
Minify: 94KB → 35KB
Gzip: 20KB → 8KB. Reducere 60% in load time. Quick win cu terser.
📜
Virtual Scroll
200 mesaje = 2000ms render. Cu virtualizare: 200ms. Render doar 15 mesaje vizibile + buffer.
🧹
Event Delegation
200+ event listeners cu 100 mesaje. Delegare pe container = 20 listeners. -85% memory.
💾
Batch localStorage
Fiecare mesaj salveaza intreg array-ul. Debounce la 3-5 mesaje = -80% writes.
🖼️
Lazy-load Features
Core (20KB) + features (30KB) lazy. Initial load: -40-50% mai rapid.
📐
Regex Simplificat
Markdown lookbehind regex prea complex. Simplificare = -50ms per 10 mesaje.
